## Key Product Lessons

### First-Principles Lesson

Repeating the First-Principles Loop is not enough. It must be used as a decision filter before design or implementation.

### DUXD Lesson

Do not abstract too early. Use real seed battlefields:

- LDD as the high-risk trading seed battlefield.
- 2026WC as the long-running prediction/reporting seed project.

### UX Lesson

A mature existing product experience should be copied before inventing. For 2026WC, the ChatGPT project page pattern was better than the custom dashboard.

### AI Board Lesson

AI Board must be visibly executed before work begins. It is not a decorative role list.

### Acceptance Lesson

Technical completion is not product acceptance. User experience acceptance decides whether a stage is accepted.
