# Tianma Work OS PRD v0.1

## Product Summary

Tianma Work OS is a multi-model AI team command system.

It helps users turn ideas, goals, problems, documents, decisions, and high-complexity scenarios into structured projects, assets, decisions, tasks, reviews, and execution support.

## Product Goals

The MVP should prove that Tianma Work OS can maintain project continuity, activate role-based AI collaboration, support source verification and risk review, convert discussions into durable documents, record decisions and review outcomes, convert pain points into DUXD requirements, and support LLM Daredevil Desk as a demanding real scenario.

## Target Users

The initial target user is a high-intensity professional user who works across complex projects, needs long-term memory continuity, generates many decisions and documents, and wants AI to act like a team.

## Core Use Cases

1. Start or resume a complex project.
2. Run AI Board Mode for a complex decision.
3. Convert pain points into requirements.
4. Generate and maintain project documents.
5. Run LDD review.

## MVP Modules

1. Project Cockpit
2. AI Board Mode
3. Project Memory and Index System
4. Decision Protocol and Decision Log
5. DUXD Requirement Pool
6. Asset Library

## LDD Requirements

User platform prices are execution source of truth. External data is for context and cross-checking. Old positions, new strategy positions, robot/grid positions, and total real-account exposure must be separated. High-risk decisions require Risk Officer review. Forecasts must be reviewed and scored.

## UX Requirements

The product should hide technical complexity. Users should experience continuity, not context management.

## Success Metrics

Project recovery speed, memory retrieval accuracy, AI Board decision quality, visibility of disagreement and risk, usefulness of Decision Log, quality of generated documents, number and quality of DUXD requirements, LDD review usefulness, reduced repeated context explanations, and user trust.
