# MVP Scope v0.1

## Purpose

The MVP should prove that AI work can become project-level, role-based, memory-aware, decision-driven, reviewable, and execution-oriented.

## Product Thesis to Validate

1. AI must move from answering questions to executing goals.
2. Serious work requires project-level memory.
3. Multi-model value comes from role collaboration.
4. Complex decisions need verification, risk review, and arbitration.
5. Product design should be driven by DUXD.
6. Domain-specific cockpits outperform generic templates.
7. Users should experience continuity, not context management.

## MVP Core Modules

1. Project Cockpit
2. AI Board Mode
3. Project Memory and Index System
4. Decision Protocol and Decision Log
5. DUXD Requirement Pool
6. Asset Library

## Included in MVP

Project Cockpit, AI Board Mode, Decision Protocol, Decision Log, Project Memory and Index, DUXD Requirement Pool, Asset Library, LDD Financial Cockpit, Source-of-truth hierarchy, Forecast review and scoring, Risk Officer role.

## Partially Included

Multi-account asset de-duplication, invisible background compression, and full multi-model routing engine.

## Excluded from MVP

Plugin ecosystem, enterprise permissions, native mobile app, fully autonomous execution, and fully automated trading execution.

## LDD as MVP Proof

LDD validates AI Board, Risk Officer, source-of-truth hierarchy, Decision Protocol, forecast scoring, project memory, asset segmentation, de-duplication, Net Exposure Map, and Position Intent Tagging.
