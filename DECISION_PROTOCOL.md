# Decision Protocol v0.1

## Purpose

The Decision Protocol defines how Tianma Work OS supports users in making important project decisions.

Important decisions must be structured, verified, reviewed, and recoverable.

## Core Philosophy

The human owns strategy and accountability. AI prepares the decision environment.

## Decision Types

Routine, Structured, Professional, High-Risk, Strategic, and Conflicted.

## Escalation Rules

Escalate to AI Board Mode if a decision is high-risk, hard to reverse, depends on fast-changing data, has source disagreement, has role disagreement, affects money/legal/safety/reputation, or changes project memory or operating rules.

## AI Board Output Structure

1. Decision question
2. Context summary
3. Known facts
4. Unknowns and assumptions
5. Role opinions
6. Consensus
7. Disagreement
8. Risk review
9. Source verification status
10. Final recommendation
11. User approval requirement
12. Decision log entry
13. Review trigger

## LDD Special Rules

1. User platform prices are the execution source of truth.
2. External data is used for cross-checking and context.
3. Old positions and strategy positions must be separated.
4. Robot/grid positions must be tracked separately.
5. No final execution instruction without current platform price when time-sensitive.
6. Forecasts must be reviewed after outcomes are known.
