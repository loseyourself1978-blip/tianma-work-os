      Fake multi-model claim

      Why it matters
      Users may overtrust simulated profiles if they look like real model routing.
      Current mitigation
      Profiles stay labeled as simulated.
      Blocking condition
      No external model is named as real without an implemented integration.

      Fake tool integration claim

      Why it matters
      Tool-looking UI can imply actual calendar, email, chat, or task access.
      Current mitigation
      Routes are marked manual route, future integration, blocked, or owner approval required.
      Blocking condition
      No integration is described as live until it exists.

      Auto-send / auto-post

      Why it matters
      Messages can create external commitments before owner review.
      Current mitigation
      All sync, email, and chat text remains manual copy only.
      Blocking condition
      No auto-send, no auto-post, and no clipboard API.

      Live trading / betting execution

      Why it matters
      Financial and betting actions are high-risk and outside the static workbench.
      Current mitigation
      LDD and 2026WC are analysis/review only.
      Blocking condition
      Execution remains blocked unless a future owner-approved milestone opens it.

      Auth / privacy / account access

      Why it matters
      Account connections expose private calendars, mailboxes, chats, files, and tasks.
      Current mitigation
      MVP-11 adds no auth and no account connection.
      Blocking condition
      Any future auth needs explicit design, consent, and fallback.

      Persistence / memory leakage

      Why it matters
      Task context can contain sensitive project details.
      Current mitigation
      All MVP-11 state is front-end-only for the current page view.
      Blocking condition
      No browser storage, backend persistence, or account sync.

      Bad acceptance gate

      Why it matters
      Weak gates let unclear work move forward as if it were accepted.
      Current mitigation
      Acceptance checks stay project-specific and visible.
      Blocking condition
      No future integration opens without a defined acceptance gate.

      Human approval bypass

      Why it matters
      TWOS should assist decisions, not replace owner approval.
      Current mitigation
      Owner approval remains central in task, route, and integration gates.
      Blocking condition
      No automatic execution path may skip explicit owner approval.
